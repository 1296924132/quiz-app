# 公文/报告润色大师

## 适用场景

- 润色申报书、调研报告、工作总结等正式公文
- 优化段落的逻辑结构、表达准确性和语言风格
- 保持原有格式（Word 文档）不变，只改进内容表达

## 核心原则

1. **先理解，再修改** — 读懂原文的核心信息后再润色，不改变原意
2. **逐句优化，给出对比** — 每条修改都要有「原文 → 改后」对照 + 修改理由
3. **风格匹配场景** — 申报书、调研报告、工作简报各有不同语言风格
4. **格式即内容** — Word 文档的排版格式（标题层级、行距、缩进）也是润色的一部分

---

## 公文写作风格对照

| 场景 | 风格要求 | 语气 | 示例词汇 |
|------|---------|------|---------|
| 申报书 | 简洁有力、条理清晰 | 客观陈述 | 旨在、着力、构建 |
| 调研报告 | 数据支撑、结论明确 | 分析性 | 数据显示、综上、建议 |
| 工作总结 | 成果导向、层次分明 | 自信务实 | 圆满完成、显著提升 |
| 通知公告 | 准确简洁、无歧义 | 官方正式 | 根据、现就、特此 |

## 润色流程

### 步骤 1：分析原文
- 确定文档类型和受众
- 找出问题段落（逻辑不清、表达啰嗦、用词不当、格式混乱）

### 步骤 2：按维度润色

#### 表达优化
| 问题 | 原文 | 改后 |
|------|------|------|
| 啰嗦 | "通过本次实践活动，使得我们队伍成员对于乡村振兴有了更加深入的了解和认识" | "本次实践活动加深了队员对乡村振兴的认识" |
| 口语化 | "我们觉得这个方案挺好的" | "该方案具有较好的可行性和适用性" |
| 模糊 | "取得了一定效果" | "问卷满意度达 92%，较实施前提升 15 个百分点" |

#### 逻辑优化
- 检查因果链是否完整（因为 A → 所以 B → 建议 C）
- 段落首句设为中心句
- 时间线要清晰（前期 → 中期 → 后期 / 现状 → 问题 → 对策）

#### 格式统一
- 标题层级用「一、→ （一）→ 1. → （1）」体系
- 数字统一（全角/半角）
- 标点符号（中文用全角，英文/数字用半角）

## 修改对照表格式

所有润色必须输出修改对照表，格式如下：

```
### 修改 1
- **位置**：第 2 段第 3 行
- **原文**：通过本次实践活动，使得我们队伍成员对于乡村振兴有了更加深入的了解和认识
- **改后**：本次实践活动加深了队员对乡村振兴的认识
- **理由**：去除冗余成分（"通过……使得……"），精简表达

### 修改 2
- **位置**：第 4 段第 1 行
- **原文**：取得了一定效果
- **改后**：活动覆盖 200+ 人次，满意度达 92%
- **理由**：用数据代替模糊表述，增强说服力
```

## 集成 Word 工作流

当润色对象是 `.docx` 文件时：

1. **读取内容**：用 `python-docx` 提取文档全文并分析结构
2. **输出修改对照表**：按上述格式展示给用户确认
3. **应用修改**：用户确认后，用 python-docx 直接修改原文档

```bash
# 提取 Word 文档内容用于分析
python -c "
from docx import Document
import sys
doc = Document(sys.argv[1])
for i, p in enumerate(doc.paragraphs):
    if p.text.strip():
        print(f'[{i}] {p.text}')
" 文档.docx
```

## 注意事项

- ⚠️ 不要改变原文的核心数据和事实
- ⚠️ 不要过度修饰导致文风不自然
- ⚠️ 修改前后长度应大致相当，不无故删减
- ✅ 优先使用主动语态，少用被动语态
- ✅ 一段话只说一个意思，不堆砌
